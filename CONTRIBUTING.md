## Getting Started

1. Fork the repository.
2. Create a new branch.
3. Make clear and well-documented commits.
4. Submit a pull request with a detailed description.

---

## Reporting Issues

- Check existing issues first.
- Open a new issue with a clear title.
- Provide reproduction steps if applicable.
- Attach logs or screenshots when helpful.

---

## Development Guidelines

- Follow the existing structure and style.
- Write clean, maintainable code.
- Avoid unnecessary files.
- Never commit secrets or credentials.

---

## Pull Request Process

Before submitting:

- Ensure the project builds correctly.
- Test your changes.
- Keep pull requests focused.
- Clearly explain what your change does.

Maintainers may request revisions before merging.
